

## 作业

我们在数据库操作的时候，比如 `dao` 层中当遇到一个 `sql.ErrNoRows` 的时候，是否应该 `Wrap` 这个 `error`，抛给上层。为什么？应该怎么做请写出代码

## 个人理解

依照搜索百度得到的对`dao`层的解释，是业务层调用的数据访问层，也就仍为它是基础库，对于基础库报的错误，在基础库对外接口中，不应当 `Wrap` 这个 `error`，而是`error`，在基础库内部解包后抛给调用层，而在调用层（也就是业务层）中进行一次`Warp`,再抛往上层，在这个error处理前不再进行`Warp`,最终在业务层顶层，进行一次错误处理，或降级，或打日志等。
